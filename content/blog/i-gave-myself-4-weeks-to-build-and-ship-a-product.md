## The idea

I wanted to build something for quite some time, but I never put enough time into it and I was always overthinking every step, especially the first one. I had no idea what to build and I found this step really hard.

I had a few ideas, but I was finding a million reasons why none of them would work. So time has passed and I was postponing a lot, up untill now where I found myself in a good spot personally and with a new boost of motivation. Now I want to capture this opportunity and build anything, so I've been thinking to just pick a product that already works and is relatively easy to build in 4 weeks. I'll have a hard deadline to ship it and that's the main goal. I won't bother with the stuff afterwards. Even if it fails I'll be learning a lot and maybe a new idea will be born in the process.

So without much thinking this time I chose to build a feature flag platform. Simple enough (at least the basic features) with quite some interesting challenges. The market looks pretty crowded and I have no idea how to eventually sell this, but that's the point, I'll think about it when and if the time comes.


## Current progress

So now as I'm writing this I'm finishing the second week and I couldn't have been happier with the progress. The whole flow works, even though it's just the barebones at the moment, I'll be polishing it in the next 2 weeks.
Currently there are 3 main components of the project: Backend API done with FastAPI, Frontend UI done with React and a python SDK. Later I'll add a Javascript SDK and no others. I'll be adding others only if needed later.
The whole thing is already deployed and I've set up CI/CD from the start so now it's really easy and enjoyable to work on this and see immediate progress.
I've been using Claude Code throughout and it's been a great fit for this kind of project. I'm a software engineer for more than 10 years and it's pretty impressive to see how far the coding models have come in the last year. I can count on the agent to do all kind of stuff especially at this scale of the project. Later on it will require a bit more manual work for sure.

I'm really excited to see where this goes. Even if it goes to the drawer soon I'm really happy with what I've learned already.
